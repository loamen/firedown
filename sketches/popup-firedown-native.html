Browser popup — Firedown-native, not Firefox copy

Stripped the Fennec-isms (tile grid, Downloads-as-library-peer) and leaned into what's already yours: the MaterialCard grouping you said you like, the 32dp icon-chip pattern from Home cards, and tinted-container colors as category signal. Pinging back to the bigger questions about Firedown identity: Downloads is your primary feature, not a library peer; Safe Folder is privacy chrome, not "your stuff"; the quick-row can hold a state toggle because Firedown's chrome is denser than Fennec's.

Firedown-native principles I'm following

  • Card-as-grouping — MaterialCard containers with surface-container background, 14dp corners (matching Home stack)
  • Tinted icon chips as the row leading element — same chip language as Home cards (Downloads = primary, Vault/Bookmarks = tertiary, time = secondary, destructive = error)
  • Downloads stays out of the popup — it's a primary chrome feature with its own bottom-bar slot and FAB. Listing it in a "library" group demotes it. Firefox can do that because Downloads is a peer in Firefox; in Firedown it's the feature
  • No tile grid — copied Firefox aesthetic that doesn't earn its place at our content volume
  • Quick-row carries ★ Bookmark — different from Fennec by choice. Firedown's quick-row already mixes transient (Share, Refresh) and stateful (Refresh re-runs JS state); the ★ as a fifth icon makes "is this page saved?" glance-able the moment the sheet opens
  • Quit is conditional — only renders when SETTINGS_QUIT_PREF is true. Rendered as a separate destructive-tinted card section, so it's visually quarantined when present

Deltas from current popup

  • Bookmark page → promoted to the quick-row as ★ (state-toggle, fills when bookmarked). Different from Fennec; justified by Firedown's denser chrome.
  • Page-state group (Find · Desktop site) gets its own MaterialCard container
  • Library group (Bookmarks · History · Safe Folder) gets a MaterialCard. Downloads excluded — already primary chrome, doesn't belong listed as a library peer
  • Icon chips appear on every row using the Home-card chip vocabulary; chip tint signals category (tertiary = saved content, secondary = time, error = destructive)
  • Settings in its own single-row card
  • Quit Firedown in its own destructive-tinted MaterialCard, only present when SETTINGS_QUIT_PREF is on. Visually quarantined — can't be hit by accident

Why this isn't a Firefox copy

  • No tile grid (that was the obvious lift)
  • Downloads deliberately excluded from popup library — a Firedown identity choice Firefox can't make
  • Star promoted to quick-row — Fennec chose otherwise; we have different chrome density
  • Tinted icon chips are your Home-card visual signature carried into the popup
