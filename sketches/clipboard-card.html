Empty-autocomplete clipboard card

When the address field is empty and the clipboard holds usable text, this card offers to use it. Phone frame with the focused address pill above each card, on the real regular + incognito surfaces, with both a URL and a plain-text clipboard to show the adaptive icon. All proposed cards use 16dp corners to match the suggestion-list container card.

Privacy note: shows the copied text while the bar is empty — same tradeoff A accepts.
Shipped: A. Fixes the 4→16dp radius mismatch, removes the duplicate search/eye icons, and shows what you'd actually paste with an icon that adapts (globe for URLs, magnifier for text). One unambiguous forward action; tapping the card or the arrow both act on the clip.

After-tap behaviour: the system clipboard is left intact (we don't clear it — that would destroy data the user copied elsewhere). The card naturally reappears only when the clipboard content changes, since it re-reads the live clip each time the empty bar is shown.

Privacy tradeoff: the old design hid the copied text behind an eye toggle; A reveals it whenever the empty URL bar is focused. Accepted deliberately for the UX win. If we ever want it back, a "masked A" (label + 'Tap to paste', content revealed only after tap) keeps the layout and the radius/icon fixes.
