Browser popup — sectioned, Fennec-inspired

Fennec uses three structural primitives instead of one flat list: a transient-actions quick-row, a page-state vertical list, and a library tile grid. The grid is the move I missed — four library destinations pack into the vertical space of two rows, become a single visual unit, and read as "your stuff" at a glance. Below: today vs proposed for Firedown's Browser popup, plus the (much simpler) Home popup.

Today vs proposed

Today's Browser popup is quick-row + one flat list of 7 mixed items (page actions, library, app). Proposed: keep the quick-row, split the list into three grouped sections — page state, library grid, app — using subtle container backgrounds (not labels) to separate them. Bookmark page stays a list row with a star icon (matching Fennec) since it's a stateful toggle that benefits from the readable row width. Library destinations compact into a 4-tile grid so the popup gets shorter while access stays direct.

What changes and why

The flat 7-row list becomes three semantic sections, each in its own rounded container. Page state = things that change THIS page. Library = places that contain your stuff. App = preferences and exit. Containers do the grouping work that labels would — same outcome, no text bloat.

  • Library grid compresses 4 destinations into ~2 rows of vertical space
  • Bookmark page stays a row but its star icon makes the toggle nature visible (fills when bookmarked)
  • Quit gets destructive tint to match its weight (mirrors Fennec's "Quit Fennec" treatment)
  • Adds Downloads to the library group — accepted mild redundancy with the bottom bar in exchange for "library lives here, all of it"
  • Bookmark/Bookmarks word-collision is gone — they're in different sections now
  • One more layout primitive (the tile grid) to maintain
  • Tile labels at narrow widths may need 10px font / abbreviated text ("Safe" instead of "Safe Folder")
